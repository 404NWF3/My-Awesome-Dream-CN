# 保研方向深挖

把本技能作为一段连续、交互式的 grill。一次只问一个主问题，等用户回答后再推进；不要把整套问卷一次性发给用户。

## 启动与续接

1. 读取 `wiki/hot.md`、`wiki/index.md` 和 `wiki/meta/preset-prompts.md`。
2. 定向查找并读取以下已有材料；不存在时跳过，不要臆造：
   - `wiki/entities/` 中 `entity_type: person` 且带 `user-self` 标签的用户实体；
   - `wiki/profile/` 中用户画像衍生页；
   - `wiki/questions/` 中 `answered_by: grill-me` 的最近记录；
   - `wiki/strategy/我的保研方向.md`。
3. 根据最近已确认并沉淀的节点判断续接位置。若无法可靠判断，直接问用户从头开始还是从哪个节点继续。
4. 没有进度时，完成读取后不要解释检查过程、空库状态或“首次运行”，不要添加标题；首次响应只输出下面这一句，文字与标点必须完全一致：

   > 你觉得你目前的保研竞争力如何（高/中/低）？

不得用画像数据替用户判定高、中、低。画像事实只用于随后校验自评和追问矛盾。

## 对话原则

- 保持直接但尊重。对口号式、互相矛盾或缺少事实的回答继续追问，直到得到可用于决策的答案。
- 每轮只推进一个节点。必要时在同一节点多追问几轮，不因拿到一句回答就机械跳转。
- 引用已有用户材料或官方信息时指出对应 wikilink；区分事实、推断和用户偏好。
- 不替用户编造绩点、排名、经历、院校门槛或截止日期。关键事实缺失时明确列为待查项。
- 除非用户选择暂停，否则持续推进到方向卡片完成。暂停时更新 `wiki/hot.md` 中的当前节点、已确认结论和下一问，并提示再次运行 `/grill-me-study` 续接。

## 决策树

### Q0：竞争力自评

- 用户选择“中/低”时走路径甲。
- 用户选择“高”时走路径乙。

### 路径甲：先挖动机

1. Q1：为什么想读研？区分学术追求、就业出路、还没想清楚/主要是惯性。
2. Q2：按回答追问：
   - 学术追求：具体研究方向、真实接触和持续投入证据；说不清时回挖动机。
   - 就业出路：目标岗位是否有硕士硬门槛，读研相对直接工作的成本收益。
   - 惯性/未想清楚：如果不读研会怎样，分开本人意愿与他人期待。
3. 汇入 Q3、Q4。

### 路径乙：先盘手牌

1. Q1：逐项盘点绩点/排名、科研、竞赛、英语、推荐人、实习与软实力，并为强弱判断给证据。
2. Q2：这副牌更适合学术导向还是就业导向；不得用“都可以”跳过取舍。
3. 仍需回到动机，随后汇入 Q3、Q4。

### 合流路径

1. Q3：愿付代价——学制、异地、院校层级、转方向不确定性，以及绝对不能接受的底线。
2. Q4：目标学位——学硕、专硕或直博，并说明为何排除另外两项。
3. Q5：地域偏好与城市硬约束；学术导向关注导师与平台，就业导向关注目标产业。
4. Q6：目标院校层——理想、主攻、保底各 1–3 所；用已 ingest 的官方门槛校验，不足则列待采集项。
5. Q7：从真实截止日期倒排时间线，标出与课业、考试或材料准备的冲突。
6. Q8：当前强弱项相对目标要求的差距，形成优势、劣势、机会、威胁与可执行补救。
7. 汇总为保研方向卡片。

## 使用预设提示词

- 在进入节点时，从 `wiki/meta/preset-prompts.md` 选择相应 `preset_slug`；按上下文改写为一个主问题，不要逐字朗读整条，也不要一次发出多条。
- 预设提示词是可选弹药，不是固定脚本。自然追问无法对应预设时使用 `prompt_type: freeform`。
- 不修改既有 `preset_slug`。提示词语义发生实质变化时使用自由提问，不伪装成旧预设。

## 关键节点沉淀

只在 Q4、Q6、Q8 和最终方向卡片节点沉淀，避免为每轮追问生成碎片页面。

1. 先向用户展示一句结论摘要，请用户纠正并按 1–5 分评价这条结论的准确/有用程度。
2. 用户确认后，在 `wiki/questions/` 创建一页；同一节点已有本轮页面时更新该页，不重复创建。页面必须包含这些顶层字段：

   ```yaml
   type: question
   title: "..."
   status: solid
   created: YYYY-MM-DD
   updated: YYYY-MM-DD
   tags: [question, grill-me-study]
   prompt: "实际提出的问题"
   question: "实际提出的问题"
   prompt_type: preset
   preset_slug: "degree-target"
   score: 4
   asked_at: YYYY-MM-DD
   answered_by: grill-me
   related: []
   sources: []
   ```

3. `prompt_type` 只能是 `preset` 或 `freeform`；自由提问的 `preset_slug` 留空。`score` 必须来自用户，不得擅自给高分。
4. 正文写清用户原答、综合结论、仍待验证的事实和相关 wikilink。保留用户已有内容，不覆盖无关字段。
5. 每次沉淀后完全重写 `wiki/hot.md`，控制在 500 个汉字以内，只保留已确认节点、当前节点、关键约束和下一问。
6. 永不修改 `.raw/`，也不把原始材料直接写进 `wiki/`。

## 完成方向卡片

在用户确认最终结论后完成三处更新：

1. 更新用户实体：保留原 frontmatter 与正文，在顶层补充或更新 `保研动机`、`目标学位`、`目标院校层`、`地域约束` 和 `updated`。
2. 更新 `wiki/profile/` 下的用户画像衍生页，使顶层 `优势`、`劣势`、`机会`、`威胁` 与 Q8 结论一致，并链接回用户实体；没有衍生页时创建 `wiki/profile/个人SWOT.md`。
3. 按 `_templates/strategy.md` 创建或更新 `wiki/strategy/我的保研方向.md`，写入目标与结论、倒排时间线、材料清单、风险预案、相关用户实体和作为依据的 question/source wikilink。

最后向用户展示一张简洁的方向卡片，至少包含：核心动机、目标学位、理想/主攻/保底院校层、地域约束、最大差距、最近三个行动、风险触发条件。明确提示在 Obsidian 中打开 `wiki/meta/dashboard.md` 查看沉淀结果。
